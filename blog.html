HTML Basics: the structural skeleton of the web

Updated Oct 9, 2025 • ~3 min read

HTML describes the semantic structure of a page. Common elements include headings <h1–h6>, paragraphs <p>, links <a>, images <img>, and sectioning elements such as <header>, <main>, <section>, and <footer>.

  • Use a single <h1> per page and nest heading levels correctly.
  • Provide meaningful alt text for images to improve accessibility.
  • Prefer semantic tags so screen readers and search engines can better understand the content.

CSS Basics: selectors and the cascade

Updated Oct 8, 2025 • ~3 min read

CSS controls visual presentation. Selectors target elements, rules declare styles, and the cascade and specificity determine the final result. Put shared styles in common_style.css, and use embedded <style> blocks or small inline overrides for page‑specific differences.

  • Common selectors: element (p), class (.note), ID (#title), attribute (img[alt]).
  • Specificity order: !important > inline style > ID > class/attribute/pseudo-class > element/pseudo-element.
  • Inline tip (inline override): reuse classes and avoid unnecessary IDs to make overrides and maintenance easier.
